## Способы запуска MCP сервера

### 1. 🚀 Рекомендуемый: Опубликованный executable (Быстро)

**Преимущества:**
- ⚡ Мгновенный запуск без компиляции
- 🎯 Оптимизированная production сборка
- 💾 Все зависимости включены
- ✅ Идеально для Claude Desktop

**Команды:**
```bash
# Один раз опубликовать
dotnet publish src/SeqMcp/SeqMcp.csproj -c Release -o ./publish

# Запускать мгновенно
./publish/SeqMcp.exe  # Windows
./publish/SeqMcp      # Linux/macOS
```

**Конфигурация Claude Desktop (Windows):**
```json
{
  "mcpServers": {
    "seq": {
      "command": "M:\\repos\\seq-mcp-server\\publish\\SeqMcp.exe",
      "env": {
        "SEQ_URL": "http://localhost:8080"
      }
    }
  }
}
```

---

### 2. 🔧 Для разработки: dotnet run (Медленно)

**Преимущества:**
- 🛠️ Удобно при активной разработке
- 🔄 Автоматическая пересборка при изменениях

**Недостатки:**
- ⏱️ Медленный старт (компиляция каждый раз)
- ❌ Не рекомендуется для Claude Desktop

**Команды:**
```bash
cd src/SeqMcp
dotnet run
```

---

### 3. ⚙️ Компромисс: dotnet run --no-build (Средне)

**Преимущества:**
- Не компилирует при каждом запуске
- Использует последнюю сборку

**Недостатки:**
- Требует ручной сборки перед запуском
- Всё равно медленнее чем опубликованный executable

**Команды:**
```bash
# Сначала собрать
dotnet build src/SeqMcp/SeqMcp.csproj -c Release

# Потом запускать
dotnet run --no-build --project src/SeqMcp/SeqMcp.csproj
```

---

## Сравнение времени запуска

| Способ | Время старта | Рекомендация |
|--------|-------------|-------------|
| **Опубликованный .exe** | ~0.5-1 сек | ✅ **Лучший для production** |
| **dotnet run --no-build** | ~2-3 сек | ⚠️ Приемлемо |
| **dotnet run** | ~5-10 сек | ❌ Только для разработки |

## Обновление опубликованной версии

После изменений в коде:

```bash
# 1. Запустить тесты
dotnet test

# 2. Переопубликовать
dotnet publish src/SeqMcp/SeqMcp.csproj -c Release -o ./publish

# 3. Перезапустить Claude Desktop (если используется)
```

## Docker (будущее)

Планируется добавить Docker контейнеризацию для ещё более простого развёртывания:

```bash
# Будущая команда
docker run -p 3001:3001 -e SEQ_URL=http://seq:5341 seq-mcp-server
```

## Рекомендации

### Для Claude Desktop пользователей:
1. ✅ Используйте опубликованный executable
2. ✅ Указывайте полный абсолютный путь
3. ✅ Настройте переменные окружения (SEQ_URL, SEQ_API_KEY)

### Для разработчиков:
1. 🛠️ Используйте `dotnet run` во время активной разработки
2. ✅ Переключитесь на опубликованный .exe для финального тестирования
3. ✅ Всегда запускайте тесты перед публикацией

### Для production:
1. ✅ Только опубликованный executable
2. ✅ Настройте мониторинг и логирование
3. ✅ Используйте systemd/supervisor для автозапуска (Linux)
4. ✅ Используйте Windows Service (Windows)
